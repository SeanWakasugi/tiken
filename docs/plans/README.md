# 作業計画

このディレクトリには、tiken を継続的に育てるための作業を、1回のセッションで完了できる粒度に分けて置きます。

## セッションの定義

1セッションは、以下をすべて満たす作業単位です。

- 目的が1つに絞られている
- 変更範囲が明確
- 完了条件を確認できる
- `npm run build` などの検証手順がある
- 次のセッションに未完了の実装判断を持ち越さない

## 各セッションの進め方

1. 該当する plan を読む
2. 作業対象ファイルを確認する
3. 実装またはドキュメント更新を行う
4. 完了条件を満たす
5. plan に必要な改善メモを追記する
