# tiken プロジェクト仕様

## 概要

tiken は、公開可能な情報を対象に、HTML / CSS / SVG / JavaScript を活用した視覚的なスライド、発表用スクリプト、図解パターン、補足ドキュメントを継続的に作成・公開・蓄積するためのナレッジ基盤です。

リポジトリ名、サイト名、設定名は `tiken` に統一します。

## 目的

- スライドをブラウザで閲覧できる HTML として作成する
- スライド本体には結論、構造、関係性、判断軸だけを置く
- 詳細説明は発表用スクリプトとして分離する
- SVG、CSSアニメーション、必要最小限のJavaScriptで図解表現を作る
- GitHubで差分管理、レビュー、再利用できる状態にする
- GitHub Pagesで公開できる静的サイトとして運用する

## 対象コンテンツ

対象:

- 公開可能な技術解説
- 一般化した開発ノウハウ
- AI活用、プロンプト設計、制作フローに関する知見
- HTMLスライドの作成方法
- インタラクティブ図解のサンプル
- 公開可能な登壇資料
- 公開可能なプロジェクト紹介
- 個人またはチームの技術メモ

対象外:

- 社内限定情報
- 顧客情報、個人情報
- 未公開プロジェクト情報
- 契約、売上、戦略に関する情報
- 社内システム構成の詳細
- セキュリティ上のリスクがある情報
- 利用許諾が不明な画像、ロゴ、フォント、データ

## 技術選定

- Astro: 静的サイト、Markdown/MDX、コンポーネント管理
- TypeScript: スクリプトとコンポーネントの型安全性
- reveal.js: HTMLベースのプレゼンテーション表示
- SVG: 構造図、関係図、フロー図、比較図
- CSS Animation: 視線誘導、状態変化、段階表示
- GitHub Actions: 自動ビルド
- GitHub Pages: 静的サイト公開

## 設計原則

1. 公開できる情報だけを扱う
2. スライドは読む資料ではなく、理解を助ける画面として設計する
3. 1スライド1メッセージを守る
4. 詳細説明は発表用スクリプトに分離する
5. 図解は装飾ではなく、構造理解のために使う
6. アニメーションは理解の順番を制御する目的で使う
7. テンプレートとコンポーネントを優先し、スライドごとの実装ばらつきを抑える
8. 公開前チェックを必ず通す

長期:

- 公開可能な技術ナレッジの蓄積場所になる
- スライド、記事、デモが一体化した発信基盤になる
- 新しい資料作成の速度と品質が安定する
