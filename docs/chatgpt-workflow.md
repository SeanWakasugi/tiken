# ChatGPT活用ワークフロー

## 位置づけ

ChatGPTは、完成物を無検証で出すためではなく、構成化、表現整理、図解案、実装補助、レビュー補助に使います。事実確認、公開可否判断、最終レビューは人間が行います。

## 標準フロー

1. 伝えたい内容を自由に話す
2. 主張、対象読者、前提、ゴールを抽出する
3. 聞き手にとって自然な順番に並び替える
4. スライドに載せる内容とスクリプトに回す内容を分ける
5. 1スライド1メッセージで構成する
6. 図解にした方がよい関係性を選ぶ
7. HTML / CSS / SVG / TypeScriptで実装する
8. 公開前チェックを行う
9. 発表後に改善点を docs に戻す

## 構成整理の依頼テンプレート

```text
以下は私が発表で伝えたい内容を話したメモです。
内容はまだ整理されていません。

あなたには、以下を行ってほしいです。
1. 伝えたい主張を抽出する
2. 聞き手にとってわかりやすい順番に並び替える
3. スライドに載せる内容と、発表用スクリプトに回す内容を分ける
4. 各スライドを「1スライド1メッセージ」で構成する
5. 文字は少なめにし、図解や関係性で伝える案を出す
6. 日本語を自然でわかりやすく補正する
7. 必要に応じて、HTML / CSS / SVGで表現できる図解案も出す

前提:
- スライドはHTMLベース
- 公開可能な情報だけを扱う
- 挿絵やイラストは不要
- 文字と図でシンプルに伝える
- 詳細な説明は発表用スクリプトに分離する

以下がメモです。
---
ここに自由に話した内容を貼る
---
```

## 実装依頼テンプレート

```text
以下のスライド構成をもとに、tiken の構成に合わせて HTML / CSS / TypeScript で実装してください。

要件:
- 1枚ごとの情報量は少なくする
- 文字と図を中心にする
- SVGで図解する
- CSSアニメーションで視線誘導する
- PCブラウザでの発表を優先する
- GitHub Pagesで公開できる静的サイトとして作る
- publicに出して問題ない内容だけにする

出力してほしいもの:
1. 変更するファイル
2. 実装コード
3. 発表用スクリプト
4. ローカル確認方法
5. 公開前チェック観点
```

## レビュー依頼テンプレート

```text
以下のスライド案をレビューしてください。

観点:
- 1スライド1メッセージになっているか
- 文字が多すぎないか
- 図解で伝えた方がよい部分はないか
- 発表用スクリプトに分離すべき内容はないか
- 聞き手にとって流れが自然か
- 公開情報として問題がないか
- タイトルや見出しが印象的か
- 動きが理解補助になっているか
- 最後に聞き手が何をすればよいか明確か

必要であれば、改善後の構成も出してください。
```
