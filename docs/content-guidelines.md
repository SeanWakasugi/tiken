# コンテンツ制作ガイド

## 基本思想

スライドは、話す内容をすべて書く場所ではありません。聞き手が話の構造を追いやすくするための補助線として設計します。

スライドに載せるもの:

- 今見てほしい結論
- 関係性を示す図
- 状態変化を示す図
- 判断軸
- キーワード
- 数字や比較
- 印象に残す一文

スクリプトに分離するもの:

- 背景説明
- 補足
- 具体例
- 注意点
- 話し言葉の説明
- 次のスライドへのつなぎ

## 1スライド1メッセージ

1枚のスライドで伝えることは1つに絞ります。

悪い例:

- AI活用の背景、課題、解決策、今後の展望を1枚に詰め込む

良い例:

- このスライドでは「AIで作る」のではなく「AIと設計する」ことが重要だと伝える

## 構造化パターン

文章だけで説明せず、以下の型に整理します。

- 入力 → 処理 → 出力
- 課題 → 原因 → 解決策
- Before → After
- 現状 → 変化 → 目標
- 表層 → 背景 → 本質
- 個別要素 → 関係性 → 全体像

## 表現設計

スライドは、既存テンプレートへ内容を流し込むのではなく、1枚ごとの「伝えたいこと」に合わせて表現方法を設計します。

テンプレートや共通コンポーネントは、余白、文字サイズ、色、ナビゲーションなどの最低限の土台に留めます。メッセージが「境界」「失敗例」「比較」「検証」「責務分担」「状態変化」などを含む場合は、その意味が画面構造として伝わるように、HTML / CSS / SVG を自由に組みます。

避けること:

- すべてを箱の列挙にする
- 関係性がないのに矢印でつなぐ
- 見た目の統一を優先して、主張の違いを消す
- 図解パターン名に内容を合わせる

優先すること:

- スライドで伝えたい判断、制約、変化、責務を先に決める
- その主張が一目でわかる画面構造を選ぶ
- 必要ならそのスライド専用のHTML/CSSを作る
- 共通化は、複数スライドで本当に同じ意味を表す場合だけ行う

## 標準スライド構成

発表資料の基本構成:

1. タイトル
2. 今日伝えたいこと
3. 背景
4. 課題
5. 課題の構造
6. 解決方針
7. 具体的な仕組み
8. Before / After
9. 実装・運用イメージ
10. 効果
11. 注意点
12. まとめ
13. 次のアクション

技術紹介スライド:

1. なぜ必要か
2. 何を解決するか
3. 全体像
4. 主要コンセプト
5. 実装方針
6. サンプル
7. メリット
8. 注意点
9. 導入手順
10. まとめ

改善提案スライド:

1. 現状
2. 問題
3. 問題が起きる理由
4. 目指す状態
5. 解決方針
6. 具体策
7. 期待効果
8. リスク
9. 進め方
10. 判断してほしいこと

## 図解パターン

よく使う図解:

- フロー図: 処理、手順、判断プロセス
- レイヤー図: システム構造、抽象度、責務分担
- 比較図: Before / After、A案 / B案
- 優先順位図: 判断ルール、重要度、リスク
- 関係図: 要素同士の関係、依存関係、影響範囲
- 状態変化図: 改善前後、時系列変化、分類変化

図解を作る順番:

1. 伝えたい結論を決める
2. 関係する要素を洗い出す
3. 要素を3から7個に絞る
4. 要素同士の関係を決める
5. 図解パターンを選ぶ
6. 余計なラベルを削る
7. 強調する要素を1つ決める
8. 必要な場合だけ動きをつける

## アニメーション

アニメーションは装飾ではなく、理解の順番を制御するために使います。

使ってよい例:

- データの流れに沿って矢印が伸びる
- 重要な要素だけが最後に強調される
- BeforeからAfterへ構造が変化する
- 複数要素が1つの結論に集約される

避ける例:

- 意味のない回転
- 常に動き続ける背景
- 過剰なズーム
- 視線を奪う装飾
- 内容理解に関係ないアニメーション
