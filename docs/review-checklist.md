# レビュー・公開チェックリスト

## 構成チェック

- [ ] 1スライド1メッセージになっている
- [ ] 発表全体の流れが自然
- [ ] 聞き手が前提知識なしでも追える
- [ ] 結論が先に提示されている
- [ ] 背景、課題、解決策、効果がつながっている
- [ ] 最後に聞き手の次の行動が明確

## 表現チェック

- [ ] 文字が多すぎない
- [ ] 見出しだけで内容の流れがわかる
- [ ] 長い説明をスクリプト側に分離できている
- [ ] 専門用語に補足がある
- [ ] 日本語が自然
- [ ] 曖昧な表現が残っていない

## 図解チェック

- [ ] 図を見るだけで関係性がわかる
- [ ] 矢印の意味が明確
- [ ] 要素数が多すぎない
- [ ] 強調箇所が1つに絞られている
- [ ] 色の意味が一貫している
- [ ] アニメーションが理解補助になっている

## 公開チェック

- [ ] 社内限定情報が含まれていない
- [ ] 顧客名、個人名、内部プロジェクト名が含まれていない
- [ ] スクリーンショットに機密情報が写っていない
- [ ] 画像、フォント、データの権利に問題がない
- [ ] 公開してよいURLやリポジトリ名になっている
- [ ] Git履歴に機密情報が残っていない

## 技術チェック

- [ ] `npm run build` が成功する
- [ ] GitHub Pagesで表示できる
- [ ] 新しいスライドがトップページと `/slides/` の一覧に自動表示される
- [ ] リンク切れがない
- [ ] スマホで最低限読める
- [ ] 発表用の画面サイズで崩れない
- [ ] reveal.js の基準サイズが `1920x1400` になっている
- [ ] SVG の `viewBox` 内部座標を不要に2倍化していない
- [ ] PDF出力時に大きく崩れない
- [ ] JavaScript無効時にも最低限内容が見える
